  • Drag over jpg files and you'll notice they can lauch into new and separate viewer elements. If that is not
    what you want, for example when only to drag and drop, just move the mouse sideways while dragging so it
    egresses the element then ingresses back into it. That will disable the auto-multi-play and only highlight.
  • "Time Of Interest" and "View Of Interest" are determined from the file last-modified (creation) time of most
    recent jpg snapshot rendered, but one minute prior to that is used as the reasonable queue-to point. Last-
    modified times of files are also used and assumed equal the exact recording end time of mp4 files...not
    always perfect equalities, but no better synchronizing method exists that can be used.
  • Javascript limitations necessitate using multiple buttons instead of just one button to perform loading then
    queueing, sorry about that. It's just that videos must get loaded enough to start playing before the queueing
    function can be executed, and only a human operator can enforce that rule. A related rule is that you may
    NOT start using page whatsoever, not even page scrolling, until entire page is completely finished rendering
    and calculating values, or unbeknownst to you, startup code will never finish initializing remaining aspects
    of the page, leaving bugs in the page's partially-loaded code.
  • Video snippets are about 6 minutes, so that is as early as the currently recording .mp4 will be ready;
    i.e.,the most recent .mp4 (at the top of the listed stream files) video will never immediately play
    since it is not finished recording. Snapshot files on the other hand, are always ready for viewing.
  • To navigate upward to the higher directory use the browser back arrow.
  • When viewing or playing by simple clicking, the back button can end the player app if the focus is on it.
  • Motion Detect snapshots often get pared down by system code in order to reduce clutter. The indication
    that happened would be an error message when trying to show the snapshot; refresh the listing.
  • I also want to be able to have this page play recordings of all cameras simultaneously & synchronized &
    play next file without intervention. That is to utilize a column to the left of the listing boxes to drop into.
    One of those columns exists as a first step of that intent.
  • My research is indicating that push notifications to Android devices might not be possible without Google's
    servers and some registration key they generate being required. I feel there should be some other way (like
    HTML5 push) that the shop server can directly trigger an Android notification. Hopefully I'll find it soon.
